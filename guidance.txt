========================================================
FINAL SYSTEM IDEA
========================================================

Project Name:
LogPulse Mini SIEM

System Description:
--------------------------------------------------------
The system receives logs from users or applications,
parses important information such as severity level,
broadcasts events in real-time using WebSocket,
stores selected events into database,
exposes metrics for Prometheus,
and visualizes metrics inside Grafana.

The system also supports:
- alerting
- severity levels
- custom rules
- Docker deployment
- CI/CD pipeline
- quality gate validation

========================================================
PHASE 1 — BACKEND DEVELOPMENT
========================================================

Objective
--------------------------------------------------------
Build the main FastAPI backend service.

Features To Build
--------------------------------------------------------
1. Health endpoint
2. Log ingestion endpoint
3. WebSocket endpoint
4. Metrics endpoint
5. Rule engine
6. Database storage

Instructions
--------------------------------------------------------

1. Create FastAPI application

2. Create:
   GET /health

3. Create:
   POST /logs

4. POST /logs should:
   - receive log message
   - parse severity
   - store to SQLite
   - broadcast via websocket
   - update Prometheus metrics

5. Create:
   WebSocket /ws

6. WebSocket should:
   - support multiple clients
   - broadcast live events

7. Create metrics endpoint:
   /metrics

8. Expose:
   - total logs
   - error logs
   - websocket clients
   - request count
   - alert count

========================================================
PHASE 2 — RULE ENGINE
========================================================

Objective
--------------------------------------------------------
Detect suspicious events automatically.

Rule Engine Flow
--------------------------------------------------------
Incoming Log
    |
    v
Parser
    |
    v
Rule Matcher
    |
    +---- Alert Trigger
    |
    +---- Severity Assignment

Instructions
--------------------------------------------------------

1. Create rules.json

2. Each rule should contain:
   - keyword
   - severity
   - description

3. Example logic:
   - "failed" => ERROR
   - "denied" => WARNING
   - "disconnect" => WARNING

4. Rule engine checks logs dynamically

5. If matched:
   - increase alert metrics
   - send websocket alert
   - optionally trigger Discord webhook
